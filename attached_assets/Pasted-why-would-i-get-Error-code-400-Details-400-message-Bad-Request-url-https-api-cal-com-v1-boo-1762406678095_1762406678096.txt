A 400 Bad Request error from Cal.com's API typically means the request format or parameters are invalid. Here are the most common reasons for this specific error:

Most Likely Issues:

Missing Required Fields - When creating a booking, you need to provide:
eventTypeId (the ID of the event type being booked)
start (ISO 8601 datetime string)
responses (object with required form fields like name, email)
timeZone
Incorrect Request Method - Make sure you're using POST for creating bookings, not GET. The URL you showed looks like a GET request.
Invalid DateTime Format - The start parameter must be in ISO 8601 format (e.g., "2025-11-05T14:00:00Z")
Missing Content-Type Header - You need Content-Type: application/json in your request headers
Invalid eventTypeId - The event type ID might not exist or might not be accessible with your API key
Example of a properly formatted request:

json
POST https://api.cal.com/v1/bookings?apiKey=cal_live_...

Headers:
Content-Type: application/json

Body:

  "start": "2025-11-06T15:00:00Z",
  "responses": {
    "name": "John Doe",
    "email": "john@example.com",
    "notes": "Looking forward to the call"
  },
  "timeZone": "America/Chicago",
  "language": "en"
}
